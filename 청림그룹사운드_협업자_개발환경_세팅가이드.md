# 청림그룹사운드 플랫폼 — 협업자 개발 환경 세팅 가이드

> 이미 구성된 저장소를 클론해서 로컬 개발을 시작하는 가이드입니다.  
> 프로젝트 초기 세팅(Supabase 프로젝트 생성, 카카오 앱 등록 등)은 이미 완료된 상태입니다.

---

## 0. 시작 전 확인사항

아래 정보를 **프로젝트 관리자(먼저 세팅한 사람)에게 미리 받아두세요.**  
환경 변수 없이는 로컬 실행이 되지 않습니다.

| 받아야 할 정보 | 어디서 |
|----------------|--------|
| `NEXT_PUBLIC_SUPABASE_URL` | Supabase 대시보드 → Settings → API |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Supabase 대시보드 → Settings → API |
| `SUPABASE_SERVICE_ROLE_KEY` | Supabase 대시보드 → Settings → API |
| `KAKAO_CLIENT_ID` | 카카오 디벨로퍼스 → 앱 → REST API 키 |
| `KAKAO_CLIENT_SECRET` | 카카오 디벨로퍼스 → 앱 → 보안 |

> ⚠️ 위 값들은 절대 카카오톡이나 메신저로 주고받지 말고,  
> **GitHub Organization의 비공개 채널 또는 직접 대면**으로 전달하세요.

## 1. 필수 도구 설치

```bash
# Node.js 20 LTS 이상
# https://nodejs.org 에서 다운로드 후 설치
node -v   # v20.x.x 이상인지 확인

# pnpm (프로젝트 패키지 매니저)
npm install -g pnpm
pnpm -v   # 설치 확인

# Git
git --version
```

> Node.js 버전이 맞지 않으면 빌드가 실패할 수 있습니다.  
> 여러 버전을 관리해야 한다면 [nvm](https://github.com/nvm-sh/nvm) 사용을 권장합니다.

## 2. 저장소 클론

```bash
# GitHub Organization 저장소 클론
git clone https://github.com/chunglim-gs/platform.git

# 프로젝트 디렉터리로 이동
cd platform
```

### 브랜치 확인

```bash
# 현재 브랜치 확인
git branch

# 개발 브랜치로 이동 (팀 브랜치 전략에 따름)
git checkout develop
git pull origin develop
```

> `main` 브랜치는 운영 배포용입니다. 개발은 반드시 `develop` 또는  
> `feat/기능명` 브랜치에서 진행하세요.

## 3. 의존성 설치

```bash
pnpm install
```

설치 완료 후 `node_modules/` 폴더가 생성됩니다.  
`package.json`이 변경될 때마다 다시 실행해야 합니다.

## 4. 환경 변수 설정

프로젝트 루트(`package.json`이 있는 위치)에 `.env.local` 파일을 새로 만듭니다.

```bash
# macOS / Linux
touch .env.local

# Windows (PowerShell)
New-Item .env.local
```

`.env.local` 파일을 열어 아래 내용을 붙여넣고, 관리자에게 받은 값을 채웁니다.

```env
# Supabase
NEXT_PUBLIC_SUPABASE_URL=https://xxxxxxxxxxxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
SUPABASE_SERVICE_ROLE_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

# 카카오
KAKAO_CLIENT_ID=카카오_REST_API_키
KAKAO_CLIENT_SECRET=카카오_클라이언트_시크릿

# 로컬 개발 URL (이 값은 그대로 사용)
NEXT_PUBLIC_SITE_URL=http://localhost:3000
```

> ⚠️ `.env.local`은 `.gitignore`에 등록되어 있어 Git에 올라가지 않습니다.  
> 실수로 커밋하지 않도록 주의하세요.

## 5. 로컬 개발 서버 실행

```bash
pnpm dev
```

터미널에 아래처럼 출력되면 정상입니다.

```
▲ Next.js 14.x.x
- Local:        http://localhost:3000
- Environments: .env.local
```

브라우저에서 [http://localhost:3000](http://localhost:3000) 을 열어 확인하세요.

## 6. 카카오 로그인 로컬 테스트 설정

로컬에서 카카오 로그인을 테스트하려면 카카오 디벨로퍼스에  
`localhost` Redirect URI가 등록되어 있어야 합니다.

**관리자에게 아래 URI 등록을 요청하거나, 직접 추가하세요.**

```
카카오 디벨로퍼스 → 내 애플리케이션 → [앱 선택]
→ 제품 설정 → 카카오 로그인 → Redirect URI → 추가

등록할 URI:
https://<프로젝트_ID>.supabase.co/auth/v1/callback
```

> `localhost:3000`을 직접 등록하는 게 아니라,  
> Supabase가 중간에서 받는 콜백 URL을 등록하는 방식입니다.  
> 이미 등록되어 있다면 이 단계는 건너뜁니다.

## 7. 내 작업 브랜치 만들기

```bash
# develop 브랜치 최신 상태로 동기화
git checkout develop
git pull origin develop

# 작업 브랜치 생성 (기능명은 영어 소문자-kebab-case)
git checkout -b feat/기능명
# 예시
git checkout -b feat/timetable-ui
git checkout -b feat/member-profile-card
git checkout -b fix/login-redirect-bug
```

### 커밋 메시지 규칙

```bash
# 형식: <타입>: <한 줄 요약>
git commit -m "feat: 합주실 주간 타임테이블 컴포넌트 구현"
git commit -m "fix: 로그인 후 리다이렉트 경로 수정"
git commit -m "chore: 환경 변수 예시 파일 업데이트"
```

| 타입 | 사용 상황 |
|------|-----------|
| `feat` | 새로운 기능 추가 |
| `fix` | 버그 수정 |
| `refactor` | 기능 변경 없는 코드 개선 |
| `style` | UI·CSS 변경 |
| `chore` | 설정·문서 등 기타 작업 |

## 9. 문제 해결 (트러블슈팅)

### `pnpm dev` 실행 시 오류가 나는 경우

```bash
# node_modules 삭제 후 재설치
rm -rf node_modules .next
pnpm install
pnpm dev
```

### 환경 변수를 찾지 못한다는 오류

`.env.local` 파일이 프로젝트 루트(package.json과 같은 위치)에 있는지 확인하세요.

```bash
# 파일 존재 여부 확인
ls -a | grep .env
# .env.local 이 보여야 합니다
```

### 카카오 로그인 후 에러 페이지로 이동하는 경우

Supabase 대시보드 → Authentication → URL Configuration에서  
`Site URL`이 `http://localhost:3000`으로 설정되어 있는지 확인하세요.  
설정되어 있지 않다면 관리자에게 추가를 요청합니다.

### `git pull` 후 패키지 관련 오류가 나는 경우

`package.json`이 변경된 경우입니다. 의존성을 다시 설치하세요.

```bash
pnpm install
```
